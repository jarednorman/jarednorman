---
name: describe-pr
description: Help the user develop a PR description for the current set of work.
---

Create a PR description that focuses on the high-level structure of the PR and aspects that will be relevant to reviewers. Do not simply replicate the information found in the commits themselves. Assume the reviewer will read those while reviewing. Instead focus on:

1. The high-level goal of the PR.
2. Information that will be helpful for reviewers to know before reading the changes.
3. Information about deploying the change.

Ensure the result is succinct and free from AI tells like em-dashes and flowery language.

Format the PR description using GitHub-flavoured Markdown.
